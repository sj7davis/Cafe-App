Two targeted edits to app/api/clock-router.ts:

EDIT 1 — Replace the getPenaltyFlag function body to convert to AEST before extracting day/hour:

Replace the current implementation with one that uses Intl.DateTimeFormat to get the wall-clock parts in "Australia/Sydney" timezone before checking day and hour. The correct approach:
- Use new Intl.DateTimeFormat('en-AU', { timeZone: 'Australia/Sydney', weekday: 'short', hour: 'numeric', hour12: false }) to get the day name and hour.
- Alternatively, create a shifted Date: const aestOffset = 11 * 60 * 60 * 1000 (AEDT) or 10 (AEST). Safe approach is Intl:
  const parts = new Intl.DateTimeFormat('en-AU', {
    timeZone: 'Australia/Sydney',
    weekday: 'short',
    hour: 'numeric',
    hour12: false,
  }).formatToParts(clockedAt);
  const dayAbbr = parts.find(p => p.type === 'weekday')?.value; // 'Sun', 'Sat', etc.
  const hourStr = parts.find(p => p.type === 'hour')?.value;
  const hour = parseInt(hourStr ?? '0', 10);

Then apply the existing penalty rules using dayAbbr and hour:
  if (dayAbbr === 'Sun') return "Sunday penalty (200%)";
  if (dayAbbr === 'Sat') return "Saturday penalty (125%)";
  if (hour >= 21 || hour < 6) return "Late night / early morning (125%)";
  return null;

EDIT 2 — Add double-clock-in guard to the clockIn mutation body, immediately after extracting staffId/venueId and before the INSERT:

  const db = getDb();
  const last = await db.select({ eventType: staffClockEvents.eventType })
    .from(staffClockEvents)
    .where(and(eq(staffClockEvents.staffId, staffId), eq(staffClockEvents.venueId, venueId)))
    .orderBy(desc(staffClockEvents.clockedAt))
    .limit(1);
  if (last[0]?.eventType === 'in') {
    throw new TRPCError({ code: 'CONFLICT', message: 'Already clocked in. Clock out first.' });
  }

The desc import is already present in the file. No other changes.
