  <name>Task 1: Create useVenueSSE hook</name>
  <files>app/src/hooks/useVenueSSE.ts</files>
  <action>
Create `app/src/hooks/useVenueSSE.ts`. This hook opens an authenticated EventSource connection to `/api/sse/orders/:venueId?token=JWT` and calls a provided `onEvent` callback when matching events arrive.

```typescript
// Hook signature
function useVenueSSE(params: {
  venueId: number | null
  token: string | null
  events: string[]                    // e.g. ['order_update', 'order_new']
  onEvent: (eventName: string) => void
}): { connected: boolean }
```

Implementation requirements:
- Use `useEffect` with `[params.venueId, params.token]` as the dependency array.
- If `venueId` is null or token is null/empty, return early without creating an EventSource. Set `connected` state to false.
- Create `EventSource` at URL `/api/sse/orders/${venueId}?token=${encodeURIComponent(token)}`.
- For each name in `params.events`, call `es.addEventListener(name, () => params.onEvent(name))`.
- On `es.open`, set `connected` state to true.
- On `es.error`, set `connected` state to false. Do NOT auto-reconnect manually — EventSource reconnects automatically.
- Return cleanup: `es.close()`.
- Export `useVenueSSE` as named export.
- Use `useRef` for `onEvent` to avoid re-creating the EventSource when only the callback reference changes (store callback in ref, call `onEventRef.current(name)` in the listener).

The `connected` boolean is returned so consumers can optionally show a connectivity indicator.
