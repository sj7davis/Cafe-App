<objective>
Harden the SSE endpoint with staff JWT authentication, and eliminate the N+1 query bug in KitchenDisplay by adding a `listOrdersWithItems` query that joins order items in a single DB round-trip.

Purpose: The unauthenticated SSE endpoint is a security blocker — any caller with a venueId receives all order data including customer PII. The N+1 bug fires one DB query per rendered KDS card, causing O(n) DB load under normal kitchen traffic.

Output: Authenticated SSE endpoint; `venue.listOrdersWithItems` tRPC query returning orders with embedded items array and tableNumber.
</objective>

    - SSE request with an expired JWT → returns 401
    - SSE request whose JWT payload venueId does not match the :venueId URL param → returns 403

In `app/api/boot.ts`, locate the SSE handler starting at `app.get("/api/sse/orders/:venueId", ...)`.

Replace the handler body with the following logic (do not alter imports or the `UPLOAD_JWT_SECRET` constant — reuse it as `JWT_SECRET`):

1. Extract token from the `Authorization` header (strip "Bearer " prefix) OR from the `?token=` query param. If neither is present, return `c.text("Unauthorized", 401)` immediately — before writing any response headers.

2. Call `await jwtVerify(token, UPLOAD_JWT_SECRET, { clockTolerance: 60 })`. Wrap in try/catch — on failure return `c.text("Unauthorized", 401)`.

3. Extract `payload.venueId` from the verified JWT payload (cast as `number`). If it does not equal the parsed `:venueId` URL param, return `c.text("Forbidden", 403)`.

4. Proceed with the existing SSE response logic: `res.writeHead(200, {...})`, `res.write(": connected\n\n")`, `addSseClient(venueId, res)`, heartbeat interval, and close handler — these lines are unchanged.

The `Authorization` header approach supports `EventSource` wrappers that can set headers. The `?token=` fallback supports native browser `EventSource` (which cannot set headers).

Note: `UPLOAD_JWT_SECRET` in boot.ts is already `new TextEncoder().encode(env.jwtSecret)` — the same secret used by staff auth. Reuse it; do not create a second constant named JWT_SECRET in this file.

In `app/api/venue-router.ts`, add a new procedure `listOrdersWithItems` immediately after the existing `listOrders` procedure (around line 267).

Input schema — identical to `listOrders` plus a `token` field for auth:
```
venueId: z.number().int().positive()
statuses: z.array(z.string()).optional()   // e.g. ['pending','confirmed','ready']
limit: z.number().int().min(1).max(200).default(100)
locationId: z.number().int().positive().optional()
token: z.string()
```

Implementation steps:
1. Verify the `token` via `jwtVerify(input.token, JWT_SECRET, { clockTolerance: 60 })`. On failure throw `TRPCError({ code: "UNAUTHORIZED" })`.
2. Verify `payload.venueId === input.venueId`. On mismatch throw `TRPCError({ code: "FORBIDDEN" })`.
3. Build `conditions` array: always include `eq(orders.venueId, input.venueId)`.
   - If `input.statuses` is non-empty: add `inArray(orders.status, input.statuses)`.
   - If `input.locationId`: add `eq(orders.locationId, input.locationId)`.
4. Fetch orders with a LEFT JOIN to orderItems in a SINGLE query using Drizzle:
```typescript
const rows = await db
  .select({
    id: orders.id,
    orderNumber: orders.orderNumber,
    customerName: orders.customerName,
    customerPhone: orders.customerPhone,
    pickupTime: orders.pickupTime,
    orderNote: orders.orderNote,
    status: orders.status,
    totalAmount: orders.totalAmount,
    tableNumber: orders.tableNumber,
    createdAt: orders.createdAt,
    itemId: orderItems.id,
    itemName: orderItems.itemName,
    quantity: orderItems.quantity,
    itemNote: orderItems.note,
  })
  .from(orders)
  .leftJoin(orderItems, eq(orderItems.orderId, orders.id))
  .where(and(...conditions))
  .orderBy(desc(orders.createdAt))
  .limit(input.limit * 20);  // over-fetch to account for multiple items per order
```
5. Group rows into `OrderWithItems[]` in JavaScript: accumulate a `Map<number, OrderWithItems>` keyed on `row.id`. For each row, push `{ itemName, quantity, note: itemNote }` into the order's `items` array when `row.itemId` is not null. Convert Map.values() to array and return.

Return type shape:
```typescript
{
  id: number
  orderNumber: string
  customerName: string
  customerPhone: string
  pickupTime: string
  orderNote: string | null
  status: string
  totalAmount: string
  tableNumber: string | null
  createdAt: Date
  items: { itemName: string; quantity: number; note: string | null }[]
}[]
```

The `staffNote` column MUST NOT appear in the output (do not include it in the select clause) — this query is consumed by KDS which runs on a shared kitchen screen.
