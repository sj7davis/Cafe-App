<objective>
Wire real Stripe Connect payments into the existing order Checkout flow. Today `stripeCheckoutRouter.createCheckoutSession` creates a plain session with no platform fee and no transfer to the venue's connected account, and the `checkout.session.completed` webhook in `boot.ts` creates a bare order row with NO line items. This plan makes every paid order route a platform application fee to the connected account and persist a complete order (with items) only after the webhook confirms payment.

Purpose: Close PAY-01 (pay before confirm), PAY-02 (confirmed + receipt on success), PAY-06 (platform fee per transaction via Connect application fees).
Output: Updated checkout session builder, webhook order persistence with items, new env binding.
</objective>
