After 08-01 / 08-02, stripeCheckoutRouter (client: trpc.stripeCheckout.*) exposes:
  createCheckoutSession.useMutation() → input { venueId, items: [{ menuItemId, itemName, quantity, unitPrice }],
    tipAmount, discountAmount, customerEmail?, customerName, customerPhone, pickupTime, orderNote?,
    locationId?, discountCode?, loyaltyPhone?, loyaltyPointsRedeemed?, metadata? } → { url, sessionId }
  createGiftCardCheckoutSession.useMutation() → { venueId, amount, senderName?, recipientName?, recipientEmail?, recipientPhone?, message? } → { url }
  createPassCheckoutSession.useMutation() → { venueId, phone, name } → { url }
  verifySession.useQuery({ sessionId, venueSlug }) → { paid, orderId, orderNumber }

Existing VenuePublic state/handlers (app/src/pages/VenuePublic.tsx):
  - handlePlaceOrder() at ~line 712 currently calls createOrder.mutate({...}) directly. Cart items are
    `cart: { menuItemId, quantity, note, modifiers }[]`; allMenuItems has { id, name, price }.
  - Discount: validateDiscountMut = trpc.promo.validateDiscount.useMutation(); appliedDiscount holds { code, description, amount }; handleApplyDiscountCode() at ~line 698.
  - Loyalty: loyaltyQuery = trpc.loyalty.getAccount.useQuery(...); loyaltyBalance = data?.pointsBalance ?? null (shown at ~line 1327: "⭐ N pts — $X available to redeem"). 10 pts = $1.
  - trpc.loyalty.getRedemptionValue exists server-side (min 100 pts) for validating a redemption amount.
  - tip/surcharge: tipAmount + surchargeAmount; effectivePromo + appliedGiftDiscount feed discountAmount today.
  - showToast(msg) helper exists.
  - Routes: customer page is /v/:slug (App.tsx). There is NO /order-status route — use ?session= query param on /v/:slug for the Stripe return (see note about success_url below).
